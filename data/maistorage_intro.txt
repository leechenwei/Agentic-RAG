PROJECT 3
Task-Oriented Dialogue System
Final Year Project
THE CHALLENGE
Most chatbots give generic answers — the goal was to build one that actually completes specific tasks like hotel booking.
KEY CONTRIBUTIONS
Built hybrid architecture: Rasa DIET classifier for fast intent recognition + LLM fallback
70% confidence threshold — only triggers expensive LLM for ambiguous queries
NER-based entity extraction to capture and map user requirements (dates, locations, etc.)
Connected to backend API (Python/Flask) for real-time data retrieval during conversation
RESULT: Cost-efficient system — ~85% of queries handled locally, LLM only for complex cases
